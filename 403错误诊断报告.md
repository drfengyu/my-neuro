# My-Neuro 403错误诊断报告

## 问题概述

虚拟桌宠在调用 LLM API 时遇到 **HTTP 403 Forbidden** 错误。

---

## 诊断过程

### 1. 初步检查
- **API地址**: https://muyuan.do/v1
- **API Key**: sk-ZWxTIw7xL0govPySm1jL... (已验证存在)
- **模型**: claude-opus-4-8
- **错误信息**: `"channel:client_restricted"` - 此通道不允许当前客户端

### 2. 代码问题分析

在 `live-2d/js/ai/llm-client.js` 第88-89行发现：

```javascript
headers: {
    'Content-Type': 'application/json',
    'Content-Length': Buffer.byteLength(postData),
    'Authorization': `Bearer ${this.apiKey}`,
    'anthropic-version': '2023-06-01',  // ❌ Anthropic专用header
    'x-api-key': this.apiKey             // ❌ Anthropic专用header
}
```

这些 Anthropic 专用的 headers 可能导致某些第三方API网关拒绝请求。

### 3. 测试结果

**测试了16种不同的User-Agent组合**，包括：
- ❌ 浏览器（Chrome、Firefox、Edge）
- ❌ Python SDK（requests、httpx、OpenAI、Anthropic）
- ❌ 开发工具（curl、Postman、Insomnia）
- ❌ 代码编辑器插件（Continue、Cursor）
- ❌ 无User-Agent

**结果**: 所有测试均返回 403，错误信息：
```json
{
  "error": {
    "code": "channel:client_restricted",
    "message": "This channel does not allow the current client (detected: xxx)"
  }
}
```

---

## 根本原因

**API通道本身设置了客户端白名单限制**，不允许任何常见的HTTP客户端访问。

这不是代码bug，而是API提供商的访问策略限制。

---

## 解决方案

### ✅ 方案1：使用云端TTS（推荐）

**已创建新启动脚本**: `Start-My-Neuro-CloudTTS.bat`

**特点**：
- 跳过本地GPU TTS服务（不需要本地TTS）
- 自动启用火山引擎云端TTS
- 支持完美的中文语音合成
- 使用"俏皮公主"音色（已配置）

**使用方法**：
```bash
双击运行: Start-My-Neuro-CloudTTS.bat
```

该脚本会自动：
1. 禁用本地TTS (`tts.enabled = false`)
2. 启用火山引擎TTS (`cloud.volcengine_tts.enabled = true`)
3. 只启动ASR和BERT服务
4. 启动Live2D前端

**配置信息**（已在config.json中）：
```json

  "enabled": true,

  "access_token": "DlShlNhCUQd19bW3S-cAsD4Yc0WpLUpj",
```

### 🔧 方案2：修复llm-client.js（可选）

如果将来需要支持更多API提供商，建议修改 `live-2d/js/ai/llm-client.js`:

**第79-92行，修改为**：

```javascript
const options = {
    hostname: url.hostname,
    port: url.port || (url.protocol === 'https:' ? 443 : 80),
    path: url.pathname,
    method: 'POST',
    headers: {
        'Content-Type': 'application/json',
        'Content-Length': Buffer.byteLength(postData),
        'Authorization': `Bearer ${this.apiKey}`
        // 移除了 anthropic-version 和 x-api-key
    },
    timeout: 60000
};
```

**说明**: 这个修改让代码遵循标准OpenAI API格式，兼容更多第三方API网关。
