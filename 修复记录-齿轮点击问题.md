# 齿轮快捷功能点击问题修复记录

**修复日期**: 2026-06-13  
**问题描述**: 齿轮快捷功能按钮（⚙）无法响应点击事件

---

## 问题分析

### 根本原因
1. **Electron 窗口鼠标穿透设置**：
   - 主进程 `main.js` 中默认设置 `win.setIgnoreMouseEvents(true, { forward: true })`
   - 窗口默认忽略所有鼠标点击事件，实现桌面宠物的"穿透"效果

2. **鼠标事件检测逻辑缺陷**：
   - `js/ui/ui-controller.js` 中的 `setupMouseIgnore()` 方法使用 **PixiJS 坐标系统**检测鼠标位置
   - PixiJS 坐标 (`global.pixiApp.renderer.plugins.interaction.mouse.global`) 与 DOM 坐标不一致
   - 当鼠标在齿轮图标上时，检测逻辑认为鼠标"不在模型上"，仍然保持 `ignore: true` 状态
   - 导致点击事件被穿透到底层窗口，齿轮的 `click` 事件永远无法触发

3. **z-index 层级冲突**（次要问题）：
   - 齿轮容器 `#quick-settings` 的 `z-index: 1001`
   - 聊天框 `#text-chat-container` 的 `z-index: 10000`
   - 齿轮被聊天框遮挡（虽然鼠标穿透是主要原因）

## 修复方案

### 1. 修复鼠标事件检测逻辑 (`js/ui/ui-controller.js`)

**问题代码**：
```javascript
setupMouseIgnore() {
    const updateMouseIgnore = () => {
        if (!global.currentModel) return;
        
        // 使用 PixiJS 坐标（错误）
        const shouldIgnore = !global.currentModel.containsPoint(
            global.pixiApp.renderer.plugins.interaction.mouse.global
        );
        ipcRenderer.send('set-ignore-mouse-events', {
            ignore: shouldIgnore,
            options: { forward: true }
        });
    };
    
    document.addEventListener('mousemove', updateMouseIgnore);
}
```

**修复后代码**：
```javascript
setupMouseIgnore() {
    let isOverUI = false;

    const updateMouseIgnore = (event) => {
        if (!global.currentModel) return;

        // 使用 DOM 鼠标坐标（正确）
        const mouseX = event.clientX;
        const mouseY = event.clientY;

        // 检查齿轮图标
        const quickSettings = document.getElementById('quick-settings');
        if (quickSettings) {
            const rect = quickSettings.getBoundingClientRect();
            if (mouseX >= rect.left && mouseX <= rect.right && 
                mouseY >= rect.top && mouseY <= rect.bottom) {
                if (!isOverUI) {
                    logToTerminal('info', `🖱️ 鼠标进入齿轮区域，设置 ignore=false`);
                    isOverUI = true;
                }
                ipcRenderer.send('set-ignore-mouse-events', { ignore: false });
                return;
            }
        }

        // 检查聊天框
        const chatContainer = document.getElementById('text-chat-container');
        if (chatContainer) {
            const rect = chatContainer.getBoundingClientRect();
            if (mouseX >= rect.left && mouseX <= rect.right && 
                mouseY >= rect.top && mouseY <= rect.bottom) {
                ipcRenderer.send('set-ignore-mouse-events', { ignore: false });
                return;
            }
        }

        // 离开 UI 区域
        if (isOverUI) {
            logToTerminal('info', `🖱️ 鼠标离开 UI 区域，恢复默认逻辑`);
            isOverUI = false;
        }

        // 默认逻辑：检查是否在模型上
        const shouldIgnore = !global.currentModel.containsPoint(
            global.pixiApp.renderer.plugins.interaction.mouse.global
        );
        ipcRenderer.send('set-ignore-mouse-events', {
            ignore: shouldIgnore,
            options: { forward: true }
        });
    };

    document.addEventListener('mousemove', updateMouseIgnore);
}
```

**核心改进**：
1. 使用 `event.clientX/clientY`（DOM 坐标）代替 PixiJS 坐标
2. 通过 `getBoundingClientRect()` 获取 UI 元素的实际屏幕位置
3. 当鼠标在齿轮或聊天框上时，显式设置 `ignore: false`，允许点击
4. 添加 `isOverUI` 状态跟踪，减少不必要的日志输出

## 技术要点

### Electron 鼠标事件机制
- `setIgnoreMouseEvents(true, { forward: true })`：窗口穿透，点击事件转发给下层窗口
- `setIgnoreMouseEvents(false)`：取消穿透，窗口可接收点击事件
- 可以通过 IPC 消息 `set-ignore-mouse-events` 动态控制

### 坐标系统对比
| 坐标系统 | 获取方式 | 适用场景 |
|---------|---------|---------|
| DOM 坐标 | `event.clientX/clientY` | DOM 元素位置检测（按钮、输入框等） |
| PixiJS 坐标 | `global.pixiApp.renderer.plugins.interaction.mouse.global` | Canvas 内的 Live2D 模型交互 |

**关键教训**：检测 DOM 元素位置时，必须使用 DOM 坐标系统，不能混用 PixiJS 坐标。

## 其他发现

### 聊天框定位问题（已修复）
- 聊天框使用复杂的"主屏幕定位"逻辑
- 在高 DPI 缩放环境下，`primaryDisplay.bounds.width` 返回缩放后的值
- 例如：2560px 屏幕在 150% 缩放下返回 1707px
- 导致聊天框可能被定位到屏幕外或错误位置
