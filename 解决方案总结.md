# 🎯 My-Neuro 虚拟桌宠 - 403问题解决方案总结

## 📋 问题确认

**现象**: 虚拟桌宠调用LLM API时返回 HTTP 403 错误  
**原因**: API通道设置了客户端白名单限制，拒绝所有常见HTTP客户端访问  
**错误信息**: `"channel:client_restricted" - This channel does not allow the current client`

1. API通道设置了严格的客户端白名单
2. 所有常见客户端都被拒绝
3. 需要联系API提供商添加白名单，或更换API

## ❓ 常见问题

### Q1: 为什么移除Anthropic headers还是403？
**A**: 因为API通道本身设置了客户端限制，不是headers的问题。

### Q2: 云端TTS需要额外配置吗？
**A**: 不需要！配置已经写好在 `config.json` 中，脚本会自动启用。

### Q3: 云端TTS会产生费用吗？
**A**: 火山引擎TTS有免费额度，正常使用不会产生费用。具体以火山引擎官网为准。

### Q4: 可以使用阿里云TTS吗？
**A**: 可以！修改脚本中的配置，将 `volcengine_tts.enabled` 改为 `false`，将 `aliyun_tts.enabled` 改为 `true`。

### Q5: 本地TTS和云端TTS可以切换吗？
**A**: 可以！使用不同的启动脚本即可：
- 云端TTS: `Start-My-Neuro-CloudTTS.bat`
- 本地TTS: `Start-My-Neuro.bat`

## 💡 技术细节

### llm-client.js 修改详情

**修改前**:
```javascript
headers: {
    'Content-Type': 'application/json',
    'Content-Length': Buffer.byteLength(postData),
    'Authorization': `Bearer ${this.apiKey}`,
    'anthropic-version': '2023-06-01',  // ❌ 移除
    'x-api-key': this.apiKey             // ❌ 移除
}
```

**修改后**:
```javascript
headers: {
    'Content-Type': 'application/json',
    'Content-Length': Buffer.byteLength(postData),
    'Authorization': `Bearer ${this.apiKey}`
    // 移除了 anthropic-version 和 x-api-key 以兼容更多API网关
}
```

**好处**:
- ✅ 标准OpenAI兼容格式
- ✅ 兼容更多第三方API网关
- ✅ 减少被拒绝的可能性
- ✅ 即使当前API仍然403，未来更换API时会更顺利
