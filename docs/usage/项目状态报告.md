## 🔴 下一步：配置 API（必需）

**在启动前必须配置 LLM API！**

编辑文件：`live-2d/config.json`

找到 `"llm"` 部分，填入以下信息：

```json
"llm": {
  "api_key": "sk-xxxxxxxxxx",           // 你的 API 密钥
  "api_url": "https://api.xxxx.com/v1/chat/completions",  // API 地址
  "model": "gpt-4",                     // 模型名称（如 gpt-4, claude-3-5-sonnet 等）
  "temperature_enabled": false,
  "temperature": 1.0,
  "system_prompt": "..."                // 已预设，一般不需要修改
}
```

### 推荐 API 服务商

   - 网址：https://www.dmxapi.cn

## ▶️ 启动项目

### 方式一：使用一键启动脚本（推荐）

```bash
# 直接双击运行
启动-My-Neuro.bat
```

脚本会自动依次启动：
1. ASR 服务（语音识别）
2. TTS 服务（语音合成）
3. BERT 服务（情感分类）
4. Live2D 主程序

### 方式二：手动逐个启动

```bash
# 第 1 步：启动 ASR 服务
1.ASR.bat

# 第 2 步：启动 TTS 服务
2.TTS.bat

# 第 3 步：启动 BERT 服务
3.bert.bat

# 第 4 步：启动主程序
cd live-2d
python start.py
```

### 方式三：使用 WebUI 控制面板

```bash
cd live-2d
# 双击运行
启动 WebUI 控制面板.bat
```

**当前状态**:

- ✅ 本地 GPU TTS 已启用 (GPT-SoVITS, 显存占用 4.2GB)
- ✅ torch 2.12.0+cpu + torchaudio 2.11.0+cpu 已安装

**性能表现**:

- ASR 语音识别: CPU 模式稳定运行
- 整体对话延迟: <2 秒（GPU TTS + 云端 LLM）

### 切换到 CPU 模式（可选）

如果显卡不可用或想节省显存，可以切换回云端 TTS:

编辑 `live-2d/config.json`:
```json
"cloud": {
  "aliyun_tts": {
    "enabled": true    // 改回 true 启用云端
  }
}
```

### 其他 GPU 用户

如果你有 NVIDIA 显卡但当前未启用:
1. 检查显卡驱动是否正常: `nvidia-smi`
2. 对于笔记本混合显卡,在厂商控制中心调整显卡模式后**重启电脑**
3. 验证 CUDA: `python -c "import torch; print(torch.cuda.is_available())"`

详细排查步骤见 `更新日志-2026-06-14-GPU支持.md`

## 📖 使用技巧

### 1. 键盘快捷键
- **V 键**：按住说话（Push-to-Talk 模式）
- **键盘打断**：随时打字打断 AI

### 2. 配置调整

编辑 `live-2d/config.json`：

```json
// 调整模型大小和位置
"ui": {
  "model_scale": 0.69,        // 模型缩放（0.1-2.0）
  "model_position": {
    "x": 0.55,                // 横向位置（0-1）
    "y": 0.25                 // 纵向位置（0-1）
  }
}

// 调整对话历史长度
"context": {
  "enable_limit": true,
  "max_messages": 18          // 保留最近 18 条消息
}
```

### 3. 更换 Live2D 模型

```bash
cd live-2d
# 扫描可用模型
一键扫描皮套动作.bat

# 在 config.json 中修改 model_type 和模型路径
```

## ❓ 常见问题

### 1. 启动后无法对话
- ✅ 检查是否配置了 LLM API
- ✅ 确认所有服务（ASR、TTS、BERT）都已启动
- ✅ 查看控制台是否有错误信息

### 2. 语音识别不工作
- ✅ 检查麦克风权限
- ✅ 确认 ASR 服务正常运行（1.ASR.bat）
- ✅ 在 config.json 中启用 ASR：`"asr": {"enabled": true}`

### 3. 没有声音
- ✅ 确认 TTS 服务正常运行（2.TTS.bat）
- ✅ 检查系统音量
- ✅ 在 config.json 中启用 TTS：`"tts": {"enabled": true}`

### 4. 端口被占用
修改对应的 .bat 脚本中的端口号，并同步修改 config.json 中的配置。
