## 快速启动（推荐）

### 方式一：使用桌面快捷方式

**首次设置**（只需一次）：

在项目根目录右键 `启动-My-Neuro.bat`，选择"**发送到 → 桌面快捷方式**"。

或者使用 PowerShell 自动创建：
```powershell
# 在 PowerShell 中运行（管理员权限）
$WshShell = New-Object -ComObject WScript.Shell
$Desktop = [System.Environment]::GetFolderPath('Desktop')
$Shortcut = $WshShell.CreateShortcut("$Desktop\My-Neuro AI 虚拟伴侣.lnk")
$Shortcut.TargetPath = "D:\Github\my-neruo\my-neuro\启动-My-Neuro.bat"
$Shortcut.WorkingDirectory = "D:\Github\my-neruo\my-neuro"
$Shortcut.Description = "My-Neuro AI 虚拟伴侣 - 一键启动"
$Shortcut.Save()
```

> 💡 记得把路径改成你实际的项目路径

**日常使用**：双击桌面图标即可启动。

---

### 方式二：直接运行脚本

```bash
# 双击运行
启动-My-Neuro.bat
```

脚本会自动依次启动：
1. **ASR 服务**（语音识别，端口 1000）
2. **BERT 服务**（情感分类，端口 6007）
3. **TTS 服务**（GPU 加速语音合成，端口 5000）*
4. **Live2D 主程序**（桌宠界面）

> \* TTS 默认使用 GPU 加速（需 NVIDIA 显卡）。如显卡不可用或想用云端 TTS，见下方"切换 TTS 方案"。

**等待时间**: 约 30-60 秒（GPU TTS 模型加载较慢，首次启动尤其明显）

## 服务端口配置

所有服务默认运行在本地：

| 服务名称 | 端口 | 完整地址 | 说明 |
|---------|------|----------|------|
| ASR | 1000 | ws://127.0.0.1:1000/v1/ws/vad | 语音识别（WebSocket） |
| TTS | 5000 | http://127.0.0.1:5000 | 本地 GPU 语音合成 |
| BERT | 6007 | http://127.0.0.1:6007/classify | 情感分类 |
| RAG | 8002 | http://127.0.0.1:8002/ask | 知识库（可选）|

## 切换 TTS 方案

### 当前配置: 本地 GPU TTS

- **优势**: 免费、离线可用、隐私、显卡加速
- **要求**: NVIDIA 显卡，约 4GB 显存
- **启动时间**: 30-60 秒（模型加载）

### 切换到云端阿里云 TTS

编辑 `live-2d/config.json`:

```json
"cloud": {
  "aliyun_tts": {
    "enabled": true,    // false 改为 true
    "api_key": "sk-xxx",  // 已配置，无需改动
    "model": "cosyvoice-v3-flash",
    // ... 其余参数
  }
}
```

保存后重启应用即可。云端 TTS 启动快（<5 秒），不占显存，但需网络和按量计费。

## LLM API 配置（重要）

**当前问题**: 现有 API key 返回 403 错误，需更换。

编辑 `live-2d/config.json`，找到 `"llm"` 部分：

```json
"llm": {
  "api_key": "sk-新的key",           // 替换这里
  "api_url": "https://新的地址/v1",  // 如需更换服务商
  "model": "claude-opus-4-8",
  "temperature_enabled": false,
  "temperature": 1
}
```

**推荐服务商**:
1. Anthropic 官方 API: https://console.anthropic.com
2. 国内中转服务（搜索 "Claude API 中转"）
3. 或联系当前服务商 muyuan.do 要一个支持 SDK 调用的 key

## 高级启动方式

### 方式一：使用批处理脚本（推荐）

一键脚本 `启动-My-Neuro.bat` 已自动化所有步骤。如需手动分步启动：

按顺序执行以下脚本：

1. **启动 ASR 服务**（语音识别）
   ```bash
   # 双击运行
   1.ASR.bat
   ```
   - 服务地址：ws://127.0.0.1:1000/v1/ws/vad
   - HTTP API：http://127.0.0.1:1000/v1/upload_audio

2. **启动 TTS 服务**（语音合成）
   ```bash
   # 双击运行
   2.TTS.bat
   ```
   - 服务地址：http://127.0.0.1:5000
   - 使用 GPT-SoVITS 模型

3. **启动 BERT 服务**（情感分类）
   ```bash
   # 双击运行
   3.bert.bat
   ```
   - 服务地址：http://127.0.0.1:6007/classify

4. **（可选）启动 RAG 服务**（知识库）
   ```bash
   # 双击运行
   RAG.bat
   ```
   - 服务地址：http://127.0.0.1:8002/ask

5. **启动 Live2D 界面**（主程序）
   ```bash
   cd live-2d
   python start.py
   # 或直接双击 live-2d/go.bat
   ```

## 配置文件

### 主配置：`live-2d/config.json`

需要配置的关键项：

1. **LLM 配置**（必需）
   ```json
   "llm": {
     "api_key": "你的API密钥",
     "api_url": "https://api.example.com/v1/chat/completions",
     "model": "gpt-4",
     "temperature": 1.0
   }
   ```

2. **服务端口**（默认配置）
   - ASR: ws://127.0.0.1:1000
   - TTS: http://127.0.0.1:5000
   - BERT: http://127.0.0.1:6007
   - RAG: http://127.0.0.1:8002

3. **功能开关**
   - `tts.enabled`: 语音合成
   - `asr.enabled`: 语音识别
   - `bert.enabled`: 情感分类
   - `rag.enabled`: 知识库
   - `vision.enabled`: 视觉识别

## 常见问题

### 1. CUDA 支持
如果要使用 GPU 加速，需要安装 PyTorch CUDA 版本：
```bash
pip install torch torchaudio --index-url https://download.pytorch.org/whl/cu118
```

### 2. 模型下载失败
可以单独下载某个模块：
```bash
cd full-hub
python Batch_Download.py --tts    # 仅下载 TTS
python Batch_Download.py --asr    # 仅下载 ASR
python Batch_Download.py --bert   # 仅下载 BERT
python Batch_Download.py --live2d # 仅下载 Live2D
python Batch_Download.py --rag    # 仅下载 RAG
```

### 3. 端口冲突
如果端口被占用，可以在各个服务的启动脚本中修改端口号。
