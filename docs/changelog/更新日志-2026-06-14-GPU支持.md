## ✅ 主要改动

### 1. NVIDIA 显卡支持修复

**问题诊断**:
- RTX 5060 Laptop GPU 硬件存在,驱动已装,但处于 `CM_PROB_FAILED_POST_START` 状态
- 原因:笔记本混合显卡(AMD 核显 + NVIDIA 独显),默认完全断电独显省电
- 机械革命控制中心的显卡模式切换需要**重启电脑**才能真正生效

**解决方案**:
1. 在机械革命控制中心调整显卡输出模式(推荐:独显直连 或 混合-自动)
2. 重启电脑
3. `nvidia-smi` 恢复正常,CUDA 12.8 可用

**验证结果**:
```
NVIDIA GeForce RTX 5060 Laptop GPU
Driver: 573.22  |  CUDA: 12.8  |  显存: 8GB
状态: OK  |  torch.cuda.is_available(): True
```

### 2. torchaudio 依赖补齐

**问题**:
- ASR 服务(FunASR)依赖 `torchaudio`,但原环境缺失
- `import funasr` 直接报错 `ModuleNotFoundError: No module named 'torchaudio'`

**解决方案**:
```bash
# torch 版本: 2.12.0+cpu (hermes-agent venv)
# torchaudio 没有 2.12 版本,最高 2.11.0

pip install torchaudio==2.11.0+cpu \
  --no-deps \
  --index-url https://download.pytorch.org/whl/cpu
```

**验证结果**:
- FunASR 成功 import
- ASR 服务正常启动,监听 1000 端口
- 语音识别链路畅通(WebSocket VAD + 上传识别)

- ✅ 每步带独立 `cd /d %~dp0xxx`,避免路径错乱

### 4. TTS 配置调整:本地 GPU 优先

**config.json 改动**:
```json
"cloud": {
  "aliyun_tts": {
    "enabled": false,        // 关闭云端,改为本地 GPU 优先
    "api_key": "sk-xxx",     // 配置原样保留作备份
    "model": "cosyvoice-v3-flash",
    // ... 其余参数完整保留
  }
}

"tts": {
  "enabled": true,
  "url": "http://127.0.0.1:5000",  // 本地 GPU TTS
  "language": "zh"
}
```

**TTS 路由优先级**(JS 按顺序判断):
1. 字节 volcengine_tts(`enabled: false`) → 跳过
2. 阿里云 aliyun_tts(`enabled: false`) → 跳过
3. SiliconFlow cloud tts(`enabled: false`) → 跳过
4. **本地 5000 端口** → ✅ 实际启用

**优势**:
- 免费、离线可用、显卡加速(合成速度快 3-5 倍)
- 云端配置整段保留,需要时改回 `enabled: true` 即可切换

## 🐛 已知问题

### LLM API 403 错误(非本次改动引起)

**现象**:
```
403 channel:client_restricted
"This channel does not allow the current client"
```

**根因**:
- LLM 服务商 `muyuan.do` 当前使用的 key/通道限制了客户端类型
- 该通道策略拒绝所有 User-Agent(浏览器、SDK、curl、空 UA 全部 403)
- 这是服务商策略问题,改代码无法解决

**影响范围**:
- ❌ 无法生成 LLM 回复
- ✅ ASR 语音识别正常
- ✅ 本地 GPU TTS 合成正常
- ✅ BERT 情感分类正常

**解决方案**:
1. 联系 `muyuan.do` 换一个允许 SDK/应用调用的 key
2. 换用其他 Claude API 提供商(官方 Anthropic / 其他中转)

## 📝 技术细节

### 混合显卡(Optimus)启用流程

1. **默认状态**:独显完全断电(`IsPresent=False`,省电)
2. **切换模式**:机械革命控制中心 → 显卡输出 → 自动/独显直连
3. **重启生效**:驱动重新初始化,独显上电(`IsPresent=True`)
4. **验证**:`nvidia-smi` 应正常显示 GPU 信息

**常见错误状态**:
- `CM_PROB_PHANTOM`:卡未上电
- `CM_PROB_FAILED_POST_START`:驱动初始化失败(需重启)

### torchaudio 版本兼容性

- torch 2.12.0 是新版,torchaudio 还没跟进到 2.12
- 用 `--no-deps` 装 torchaudio 2.11,不动 torch 2.12
- 实测兼容性 OK(FunASR 正常加载和识别)
- 若后续出现异常,用 `pip install torch==2.11.0+cpu torchaudio==2.11.0+cpu` 对齐版本
